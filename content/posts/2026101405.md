## 流式输出（SSE）也不怕

流式输出才是最体现“省事”的地方。手动解析SSE数据，要处理`data: `前缀、空行、`[DONE]`标记等一大堆细节。

用我给你的POJO结构，加上Jackson的流式反序列化，代码依然干净：

java

// 流式示例核心片段
HttpRequest streamingRequest = HttpRequest.newBuilder()
    .uri(URI.create(BASE_URL + "/chat/completions"))
    .header("Authorization", "Bearer " + API_KEY)
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(
        MAPPER.writeValueAsString(new ChatRequest("qwen3-7b", 
            List.of(new ChatMessage("user", "讲个笑话")), true))
    ))
    .build();

client.sendAsync(streamingRequest, HttpResponse.BodyHandlers.ofLines())
    .thenAccept(response -> {
        response.body().forEach(line -> {
            if (line.startsWith("data: ") && !line.equals("data: [DONE]")) {
                try {
                    // 直接解析SSE数据块
                    ChatResponse chunk = MAPPER.readValue(line.substring(6), ChatResponse.class);
                    if (chunk.choices() != null && !chunk.choices().isEmpty()) {
                        System.out.print(chunk.choices().get(0).delta().content());
                    }
                } catch (Exception e) {
                    // 略过
                }
            }
        });
    });
