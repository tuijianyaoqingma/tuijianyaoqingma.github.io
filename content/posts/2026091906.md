## 第四步：开始对话——流式输出这个坑一定要看

很多人都希望模型像打字机一样，一个字一个字的往外吐，这叫**流式输出**。这在体验上很重要，尤其是当你写对话机器人或实时生成内容时。

但新手很容易写错。下面是正确的流式写法：

python
stream = client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=[{"role": "user", "content": "写一首关于秋天的短诗，每行10个字以内。"}],
    stream=True  # 开启流式
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")


**避坑第五条**：流式模式下，`response`对象不再是一个完整的消息包，而是一个迭代器。很多新人还在用`response.choices[0].message.content`去拿数据，结果只拿到了`None`。切记：流式模式下，内容存在于`delta.content`字段，你需要一片一片的拼起来。

**避坑第六条**：`delta.content`在流式推送过程中，有可能为`None`（比如在结束信号里）。所以一定要加上`if chunk.choices[0].delta.content is not None:`的判断，不然会报`NoneType`错误。

## 第五步：处理异常——别让报错把你劝退

码农最怕的，不是Bug，而是莫名其妙的500错误。下面我给你一个健壮的调用模板，把这些常见的坑全给你堵死：

python
from openai import OpenAI, APIError, APIConnectionError, RateLimitError
import time

client = OpenAI(
    api_key="sk-你的云雾apikey",
    base_url="https://www.yunwu.ai/v1"
)

def call_gpt41mini(user_prompt, retries=3, delay=2):
    for attempt in range(retries):
        try:
            response = client.chat.completions.create(
                model="gpt-4.1-mini",
                messages=[{"role": "user", "content": user_prompt}],
                temperature=0.5,
                max_tokens=1024
            )
            return response.choices[0].message.content.strip()
        except RateLimitError:
            print("速率限制，正在重试...")
            time.sleep(delay * (attempt + 1))
        except APIConnectionError:
            print("网络连接错误，检查网络或代理设置...")
            time.sleep(delay)
        except APIError as e:
            print(f"API错误：{e}")
            time.sleep(delay)
    return "调用失败，请检查日志。"

# 调用测试
result = call_gpt41mini("请列举5个Python的最佳实践。")
print(result)


**避坑第七条**：遇到`429 Too Many Requests`或`500`错误时，不要无脑一直请求。加一个指数退避（Exponential Backoff）的重试机制。上面的代码已经帮你实现了。

## 第七步：高级技巧——参数调优与成本控制

别用一个`temperature=0`跑天下。GPT-4.1mini对各种参数很敏感：

- **temperature（创造力）**：做代码生成或数学推理时，建议设为0.1至0.3；写文案或故事时，0.7至0.9更合适。
- **max_tokens（输出长度）**：为了省钱，每次调用前想清楚你需要的最大输出量，不要默认4096。例如做摘要，设置`max_tokens=256`就够了。
- **top_p（核采样）**：通常和temperature二选一调整。不推荐同时都调，建议先只调temperature。
