## 报错大全：你这辈子在接入路上会碰到的所有坑

我知道很多人最怕的就是配置完成后一直报错。我把自己摸索过程中踩爆的几个坑都写在下面了，直接对症下药。

### 错误 1：403 Forbidden / Unauthorized

**原因**：API Key 没传对，或者 Key 过期了。

**解法**：检查你的 Header 里 Authorization 字段。它必须长这样：

`Authorization: Bearer 你复制的完整KEY`

千万不要在前面加“key=”、“token=”或者用其他格式。 Open AI 库默认会处理好格式，手动写 curl 时要特别注意。

### 错误 2：404 Not Found：Model not found（模型不存在）

**原因**：模型名称没写对，云雾里不叫“gpt4-5.1-codex”，而是可能有空格、大小写、版本号不一样。

**解法**：去后台的模型列表里直接复制 `gpt-5.1-codex` 这个字符串。

### 错误 3：429 Too Many Requests（请求过多）

**原因**：并发太高，超过了限额。云雾免费额度阶段会有并发限制。

**解法**：
1. 在代码里加 `time.sleep(1)`，每次请求间隔1秒。
2. 如果需求量大，直接充值（最低1块），并发限制就自动解除了。

### 错误 4：Connection Timeout （连接超时）

**原因**：你的网络环境或者服务器在国内，但 API 请求直接到了海外 IP。确保没有用旧版 OpenAI SDK。

**解法**：确保 base_url 填的是云雾的 `https://www.yunwuai.cc/v1`，而不是 `https://api.openai.com`。否则国内网络会直接超时。

### 错误 5：Token 长度超限

**原因**：GPT-5.1-Codex 虽然是强模型，但上下文长度也有上限。

**解法**：
- 传参时加上 `max_tokens` 限制。
- 可以分段发送长内容。
- 在调用参数里设置 `truncate` 为 `true`（部分模型支持截断而不是报错）。

### 错误 6：Received invalid JSON response

**原因**：网络中间层有杂音或者代理干扰。可能你仍在某个海外中转。

**解法**：关闭所有 VPN 和代理，让请求直连国内云雾服务器。
