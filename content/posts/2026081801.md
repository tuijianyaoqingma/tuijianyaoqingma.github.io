云雾的接口完全兼容 OpenAI，所以你只需要安装官方库就行。如果你还没装，跑一行命令：

这一步是最关键的。云雾给你生成 API Key 之后，你需要在代码里设置两个东西：

- `api_key`: 你在云雾后台申请到的密钥

看代码：

python
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的云雾API密钥",  # 别硬编码！后面会讲密钥管理
    base_url="https://www.yunwuai.cc/v1"

    ]

print(response.choices[0].message.content)


跑一下，如果返回了排序算法的 Python 代码，说明接入成功了。

### 3. 非流式 vs 流式输出

o3 的推理过程比 GPT-4 长，用流式（streaming）能让用户体验更好，不用等全部生成完才能看到结果。

非流式调用就是上面的代码，请求发出去，拿到完整响应才打印。

流式调用需要加一个 `stream=True` 参数：

python
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的密钥",
    base_url="https://www.yunwuai.cc/v1"
)

stream = client.chat.completions.create(
    model="o3-mini",
    messages=[{"role": "user", "content": "用 Python 写一个冒泡排序"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")


实际体验中，流式输出速度非常快。云雾的节点覆盖全球七大地区（美国、日本、韩国、英国、香港、菲律宾、俄罗斯），据说直连速度是官方 API 的 1200 倍（AZ 企业级通道加持）。我没测到那么夸张，但确实比挂代理去调 OpenAI 原生接口快不少。

## 密钥管理避坑指南：这个坑我替你踩过了

写代码调 API 很容易，但把密钥安全地管理好，就不那么简单了。下面几个坑，都是我自己在实际项目里碰到过的。

### 踩坑 1：直接把 API Key 写在代码里

这是最常见的问题。很多人图方便，直接把密钥写死在 Python 文件里，然后上传到 GitHub 公开仓库——结果第二天发现密钥被刷光了，账单上多了一大笔钱。

**解决方案：** 永远用环境变量。

python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("YUNWU_API_KEY"),
    base_url="https://www.yunwuai.cc/v1"
)


然后在终端里设置环境变量：

bash
export YUNWU_API_KEY="sk-你的云雾API密钥"


如果你用的是 Windows PowerShell：

powershell
$env:YUNWU_API_KEY="sk-你的云雾API密钥"


### 踩坑 2：用错了密钥格式

云雾生成的密钥格式是 `sk-` 开头的一串字符。有些人从别的地方复制错了，或者用了别的平台的密钥来调云雾接口，结果一直报 401 认证错误。

**解决方案：** 登录云雾后台，到 API 管理页面查看你的密钥。确保复制完整，没多空格、没缺字符。如果你不确定，可以重新生成一个试试。

### 踩坑 3：密钥泄露了不知道

如果你的密钥不小心泄露了，云端被人刷了，你可能会在第二天才发现——那时候已经损失惨重了。

**解决方案：** 定期轮换密钥。云雾支持随时生成新密钥，同时让旧密钥失效。建议每个月换一次，或者在怀疑泄露时立即更新。在代码里，可以用多个密钥做负载均衡和容灾，但密钥管理要统一，用一个环境变量数组来存。

python

api_keys = os.environ.get("YUNWU_API_KEYS", "").split(",")

client = OpenAI(
    api_key=selected_key,
    base_url="https://www.yunwuai.cc/v1"

### 踩坑 4：Base URL 写错了

有人把 `base_url` 写成了 `https://api.openai.com/v1`，然后用自己的 OpenAI 密钥去调——当然不行。也有人写成了 `https://www.yunwuai.cc/` 少了个 `/v1`，结果接口报 404。

**解决方案：** 记住，云雾的接口完整地址是 `https://www.yunwuai.cc/v1`。写代码时务必确认路径正确。
