## 核心操作：在小程序后端中接入API

假设你已经有一个小程序的后端（Node.js、Python、Java等），或者你打算使用微信云的云函数（基于Node.js）。我以最常用的 **Node.js + 云函数** 为例，展示如何对接。

### 1. Node.js 后端接入代码 (Express示例)

你的后端需要暴露一个接口给小程序前端调用。

javascript
const express = require('express');
const axios = require('axios'); // 推荐使用axios，支持流式响应
const app = express();

// 你的云雾API Key
const YUNWU_API_KEY = '你的云雾API Key'; // 记得从环境变量读取，别写死在代码里！

app.post('/api/chat', async (req, res) => {
  const { messages } = req.body; // 从小程序发来的对话历史

  try {
    const response = await axios.post('https://www.yunwuai.cc/v1/chat/completions', {
      model: 'gpt-3.5-turbo', // 或者用 'gpt-4o', 'claude-3-haiku' 等
      messages: messages,
      temperature: 0.7,
      stream: false, // 小程序端用非流式更简单
    }, {
      headers: {
        'Authorization': `Bearer ${YUNWU_API_KEY}`,
        'Content-Type': 'application/json',
      },
      timeout: 30000 // 30秒超时，防止请求卡死
    });

    // 返回结果给小程序
    res.json({
      success: true,
      data: {
        reply: response.data.choices[0].message.content
      }
    });

  } catch (error) {
    console.error('API调用失败:', error.response?.data || error.message);
    res.status(500).json({
      success: false,
      error: 'AI服务暂时繁忙，请稍后再试'
    });
  }
});

app.listen(3000, () => console.log('服务已启动'));

## 常见问题与避坑指南

**问题1：调用失败，返回401 Unauthorized**

`错误原因`：你的API Key写错了，或者没有在请求头里正确设置。
`解决方案`：检查你的`Authorization`头，格式必须是 `Bearer 你的密钥`。去[云雾ai聚合站后台](https://www.yunwuai.cc/register?channel=c_7o7g8tlk)重新复制并检查密钥。

**问题2：响应很慢，或者经常超时**

`错误原因`：选择了过于复杂的模型（如GPT-4），或者网络状况不佳。
`解决方案`：
- 对于聊天、问答场景，优先使用`gpt-3.5-turbo`或`gpt-4o-mini`，速度快、成本低。
- 在小程序请求中增加`timeout`设置（如上所示）。
- 如果使用云函数，增加云函数的超时时间到10秒以上。

**问题3：如何选择模型？**

| 场景                 | 推荐模型            | 特点                     |
| ---------------------- | --------------------- | -------------------------- |
| 简单问答、客服机器人 | `gpt-4o-mini`       | 速度快，成本极低         |
| 需要深度推理、代码生成 | `gpt-4o`            | 聪明，但速度稍慢         |
| 中文长文本处理       | `deepseek-v2`       | 中文优化好，性价比高     |
| 生成创意文案         | `claude-3-haiku`    | 对创意和审美理解能力强 |

**问题4：流式响应（Stream）如何实现？**

对于小程序这种场景，强烈建议**不要用流式**。小程序对`wx.request`的限制比较多，处理流式数据很麻烦。直接用非流式请求（即`stream: false`），拿到完整数据后再渲染UI，体验更稳定。
