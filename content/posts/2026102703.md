## 避坑清单：100% 不报错的精排经验

我见过太多次“代码看起来很完美，但就是跑不通”的场景。下面这几条，每一条都可能卡住你一次。记下它们，你就能一次性搞定，省去怀疑人生的时间。

### ❌ 坑 1：模型名写成 `qwen-plus` 或 `QWEN-PLUS`

**正确写法：`Qwen-Plus`**（大写开头，大小写敏感）

就这么一个大小写区别，多少倒霉蛋的脚本直接弹 "model not found"。从云雾平台拿到的官方模型列表，老老实实复制粘贴过去。

### ❌ 坑 2：忘记改 `api_base`

这是个老毛病，有人复制网上教程里 OpenAI 的代码，没把 URL 换成 `https://www.yunwuai.cc/v1`。改了这里，你的请求才能绕过代理限制。

### ❌ 坑 3：API Key 包含多余空格或引号

有时候你会直接从短信或邮件复制密钥，前面或者后面不小心粘了一个空格。建议写成：

python
openai.api_key = "sk-你的密钥ID".strip()


强制去空格，心里舒服。

### ❌ 坑 4：先写 `stream=True` 然后不知道怎么办

**新手别碰流式模式。** 直接用我们上面的 `stream=False`。

很多教程为了酷炫写生成器样式，但你对接平台如果不懂反正要做一番过滤、拼接响应，大概率直接当场崩溃。先跑通非流式，再玩花的。

### ❌ 坑 5：代理与网络设置冲突

用了某些公司的内网代理，或者开了翻墙工具，一旦和云雾的中转地址冲突会报 SSL 错误。要不关掉代理跑，或者学习配置 `proxies`：

python
openai.proxies = {"http": None, "https": None}


### ❌ 坑 6：用太旧的 `openai` 库版本

`openai` 库的 API 调用方式几年内变化不少。确保版本是 0.28 以上，或者干脆 `pip install openai --upgrade`，减少兼容性问题。

### ❌ 坑 7：随意的 `max_tokens` 不给控制

如果有人给你一段无限对话又没设 `max_tokens`，一不小心就烧掉一大堆额度。先设置一个低值 512 或 1024。
