## 第三步：配置云函数（或自己写服务器）

飞书机器人需要一个后端来处理消息。有两种常见方案：
1. **用云函数**（如阿里云函数计算、腾讯云函数）：便宜且无服务器，适合个人测试。
2. **自己买个轻量服务器跑代码**：稳定但贵一些。

我这次用的是阿里云函数计算（FC），代码逻辑很简单：接收飞书发来的消息，解析用户输入的文本，然后调用云雾的API接口，得到回复后返回给飞书。

核心代码大概长这样：

python
import openai
import json

# 关键：改成云雾的地址
openai.base_url = "https://www.yunwuai.cc/v1"
openai.api_key = "你的云雾API Key"

def handle_message(event):
    # 解析飞书消息
    user_input = event['message']['content']
    # 调用大模型

        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": user_input}]
    )
    reply = response['choices'][0]['message']['content']
    return reply


注意看上面代码里最关键的那两行：`base_url`改成云雾的地址，`api_key`换成你的key。这样写出来的代码，实际上调用的是云雾提供的直连通道，后面不需要翻墙。
