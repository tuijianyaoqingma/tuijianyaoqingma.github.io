1.  打开飞书开放平台（open.feishu.cn），点击“创建应用” -> “企业自建应用”。

这一步跟普通飞书开发一样，只是我们要在后端加上AI。

### 第三步：在你的服务器上部署一个轻量化的中间件

飞书机器人不会直接调用外部API，需要你有一个服务器（比如一台廉价的VPS、云函数、甚至是你自己的电脑）来架设一个中间件。中间件接收飞书的消息，调用AI的API，再把结果返回。

from flask import Flask, request, jsonify
import requests, json

app = Flask(__name__)

# 你的云雾API Key
API_KEY = "sk-你的API_KEY"
API_URL = "https://www.yunwuai.cc/v1/chat/completions"

@app.route('/webhook', methods=['POST'])
def webhook():
    # 解析飞书推送的消息（这里简化了签名验证）
    data = request.json
    msg_content = data.get('event', {}).get('text', '')
    
    # 构造请求体（兼容OpenAI格式）
    payload = {
        "model": "gpt-4o-mini",
        "messages": [{"role": "user", "content": msg_content}],
        "stream": False
    }
    headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"}
    
    # 真正的调用：直接走云雾API，无需任何代理

    # 返回给飞书
    return jsonify({
        "msg_type": "text",
        "content": {"text": ai_reply}
    })

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=False)

## 接入过程中可能踩的几个坑（以及避坑指南）

*   **问题：服务器访问云雾API超时**
    *   解决：检查服务器前端防火墙。云雾提供的API接口能直接国内访问，但你的服务器可能限制了对外出站，放开443/80端口试试。

    *   解决：一定要把飞书开放平台的“回调配置”中的`Verification Token`和`Encryption Key`正确配置到代码里，否则飞书会认为请求不合法。

    *   解决：确认返回给飞书的内容字段是 `{"msg_type":"text", "content":{"text":"..."}}`。
