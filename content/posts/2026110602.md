说实话，我第一次调这种模型的时候，也卡了将近半小时。问题基本出在这些地方，我一并给你列好，避免你再陷入怀疑：

### 1. `baseURL` 写错了
是的，很多老司机这点也容易错。如果你的代码一直在报 404 或者 connection refused，大概率是 `baseURL` 结尾没加 `/v1`，或者是写成了其他格式。记住，必须是：

https://www.yunwuai.cc/v1

不要多想，直接抄作业。

### 2. API Key 里混进了多余的空格
你复制 API Key 的时候，可能会把前后空白字符也一起带上了。在前后加个 `trim()`，或者直接手动检查粘贴后有无多余空格，这个小问题能省你半小时。

### 3. 模型名写错成 “grok-3-mini”（实际该用小写）
Grok3mini 在[云雾ai聚合站](https://www.yunwuai.cc/)的模型列表中名称就叫 `grok-3-mini`，全小写加连字符。不要自己发明名字，写 `Grok3mini` 或者大写开头都会报错。

### 4. 网络锁死或防火墙问题
虽然云雾可以国内直连，但有些公司内部的网络可能把一些 API 端口的请求给拦截了。如果你在公司环境里的代码跑不通，建议用家里的网络，开着手机热点试一下先排除问题。
