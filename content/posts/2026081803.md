## 常见报错和解决办法

很多人改完 baseurl 还是报错，我总结一下新手最容易踩的坑：

| 报错信息                    | 原因                             | 解决办法                                       |
| ----------------------------- | ---------------------------------- | ------------------------------------------------ |
| `401 Unauthorized`          | API key 不对或没传                | 检查是否用了云雾的 API key，不是 OpenAI 的 key |
| `404 Not Found`             | baseurl 末尾多了一个斜杠或格式错 | 确保 baseurl 是 `https://www.yunwuai.cc/v1`   |
| `Connection Timeout`        | 网络问题，或 baseurl 写错了      | 重新检查 baseurl 拼写，确保国内网络环境下直连 |
| `403 Forbidden`             | 账户余额不足或模型未授权         | 充值 1 元就能解决，检查账户余额                   |
| `Invalid model`              | 模型名写错或云雾不支持           | 去官网看支持模型列表，确认拼写                  |

如果你遇到 `404` 问题，**90% 是因为 baseurl 末尾多了 `/`**。这是最常见的低级错误，但也是最容易修好的。
