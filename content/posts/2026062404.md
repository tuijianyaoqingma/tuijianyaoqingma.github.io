## 第三步：内网环境部署的避坑指南

这一步是企业接入中最容易翻车的环节。很多团队在内网环境下调用云雾api，出现超时、连接失败等问题，多半是网络配置没搞定。

**核心配置要点：**

### 1. 确认内网能访问公网

[云雾api中转站](https://www.yunwuai.cc/)的地址 `https://www.yunwuai.cc/v1` 是公网地址，内网服务器需要能通过 NAT 或代理访问公网。如果你的服务器完全不联网（纯物理隔离），那就没法用任何在线 API，包括云雾。

✅ **解决方案**：
- 通过内网代理服务器（如 Squid、Nginx 正向代理）转发请求
- 在内网 DNS 中添加 `www.yunwuai.cc` 的公网解析，避免 DNS 劫持
- 建议在代码中设置 `timeout` 超时时间（例如 10 秒），避免因网络波动导致进程卡死

python
# 设置超时时间
openai.timeout = 10  # 单位：秒


### 2. 处理 SSL/TLS 证书问题

少数内网环境会因为 CA 证书不完整导致请求失败。

✅ **解决方案**：
- 在代码中禁用 SSL 验证（仅限调试用，生产环境不建议）

python
import ssl
import certifi

# 使用 certifi 提供的 CA 包，而不是依赖系统默认
openai.request_options = {
    "verify": certifi.where()
}


- 或者直接指定证书路径，推荐使用 certifi 库自动处理

### 3. 请求频率与并发控制

[云雾api中转站](https://www.yunwuai.cc/)支持无限制并发，但如果你在内网环境中同时发起大量请求（比如 1000 个并发），可能会被对方限流，或者导致内网带宽打满。

✅ **解决方案**：
- 建议使用 **请求队列 + 重试机制**
- 如果遇到 429 限流错误，添加 `retry` 逻辑，等待 1-2 秒后重试
