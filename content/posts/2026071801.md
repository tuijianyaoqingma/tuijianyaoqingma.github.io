### 第三步：请求Kimi模型，获取稳定回复

java
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.Arrays;

public class KimiDemo {
    public static void main(String[] args) {
        String apiKey = System.getenv("KIMI_API_KEY");
        OpenAiService service = new OpenAiService(apiKey, Duration.ofSeconds(60));
        // 指定模型为Kimi的模型ID（具体ID在云雾控制台可查）
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("kimi") // 这里填入云雾支持的Kimi模型ID
                .messages(Arrays.asList(
                        new ChatMessage("system", "你是一个专业的Java助手。"),
                        new ChatMessage("user", "请用Java写一个斐波那契数列的递归实现，并解释优化方式。")
                ))
                .maxTokens(1024)
                .build();

        service.createChatCompletion(request).getChoices().forEach(choice -> {
            System.out.println(choice.getMessage().getContent());
        });
    }
}


你看，上面就是完整的接入代码，核心变动只有一行——改`baseUrl`。这行改动意味着：你的网络请求不再直连Kimi海外服务器，而是走[云雾ai大模型中转站](https://www.yunwuai.cc/)的国内高速链路。一旦网络有问题，中转站会自动重试3次，不会直接断连返回错误码。跑完这个示例，你就能体会到什么叫**5分钟稳定调通**。
