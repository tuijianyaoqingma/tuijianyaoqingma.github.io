在你的`service`或者`utils`包下，新建一个类叫`AiService`或者`GptService`，然后写下面这样一段代码：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.ArrayList;
import java.util.List;

public class GptService {

    private static final String API_KEY = System.getenv("YUNWUAI_API_KEY"); // 从环境变量读取Key
    private static final String BASE_URL = "https://www.yunwuai.cc/v1"; // 这是唯一需要你改的地方

    public static String callGpt41(String userMessage) {
        // 1. 创建OpenAiService实例，只需要传baseUrl和key
        OpenAiService service = new OpenAiService(BASE_URL, API_KEY);

        // 2. 构建请求消息
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage("system", "你是一个编程助手，用Java风格回复。"));
        messages.add(new ChatMessage("user", userMessage));

        // 3. 构造请求参数
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-4.1") // 这里指定模型名，[云雾ai聚合站](https://www.yunwuai.cc/)支持gpt-4.1
                .messages(messages)
                .maxTokens(1000)
                .temperature(0.7)
                .build();

        // 4. 发起调用并返回结果
        return service.createChatCompletion(request)
                .getChoices()
                .get(0)
                .getMessage()
                .getContent();
    }

    public static void main(String[] args) {
        // 测试：生成一段Java代码
        String query = "用Java写一个单例模式的类";
        String result = callGpt41(query);
        System.out.println(result);
    }
}

## 其他技巧：怎么切换模型、怎么处理错误

上面的代码只是最基础的三步。实际项目里，你可能需要做更多细节处理。

**切换模型：**
在`ChatCompletionRequest.builder().model()`方法里，把`gpt-4.1`换成任何[云雾ai聚合站](https://www.yunwuai.cc/)支持的模型名称就行。比如换成`gpt-4o`、`claude-3-opus-20240229`、`deepseek-r1`。重点是——你不需要改动其他任何代码，模型由URL统一管理。想比较不同模型的效果？写一个循环，依次调用不同model名称就完事了。

**错误处理：**
OpenAI Java SDK里封装了异常体系。比如最常见的`HttpException`，可能因为网络问题或API Key过期导致。建议统一在调用层加一个try-catch：

java
try {
    String result = callGpt41("你的问题");
} catch (Exception e) {
    // 打印异常信息，或重试逻辑
    System.err.println("API调用失败: " + e.getMessage());
}


[云雾ai聚合站](https://www.yunwuai.cc/)的稳定性官方标称99.9%，但任何网络服务都有极低概率失败，建议项目中实现指数退避重试（exponential backoff），加一个简单的循环重试三次。
