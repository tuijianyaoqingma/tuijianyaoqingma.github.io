### 第二步，写核心调用代码

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.Arrays;

public class Qwen3Client {
    public static void main(String[] args) {
        // 关键行：base_url 写成云雾API的地址
        String baseUrl = "https://www.yunwuai.cc/v1";
        String apiKey = "sk-xxxx"; // 换成你在云雾申请的key

        OpenAiService service = new OpenAiService(apiKey, Duration.ofSeconds(30));
        service.setBaseUrl(baseUrl);

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("Qwen3-70B-Plus")  // 模型名由云雾提供，Qwen3已经在列
                .messages(Arrays.asList(
                        new ChatMessage("system", "你是一个简短回复的助手，每次回复不超过50个Token。"),
                        new ChatMessage("user", "请用50字以内概括Java的垃圾回收机制。")
                ))
                .maxTokens(60)  // 精确控制响应长度，实现零浪费
                .build();

        service.createChatCompletion(request).getChoices().forEach(choice -> {
            System.out.println(choice.getMessage().getContent());
        });
    }
}


看到没？我故意在system prompt里加了一句“每次回复不超过50个Token”，又把`maxTokens`设置成60。这么做是为了保证：如果模型跑偏了，响应直接被截断，不会多花Token。而且Qwen3本身在云雾的调用成本就极低，加上这个控制，基本不会产生浪费。
