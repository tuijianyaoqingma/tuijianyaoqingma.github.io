3. **没有使用流式**：如果后续你要做响应处理，非流式会占用更多内存和网络开销。

### 优化后省钱版（每次省0.02元）

python
from openai import OpenAI
import time

client = OpenAI(
    api_key="你的云雾API_KEY",
    base_url="https://www.yunwuai.cc/v1"
)

# 关键：在messages中隐藏模型参数，或通过自定义header指定分组
# 云雾支持在请求头添加 X-Server-Name 选择分组
# 这里演示用stream=True, max_tokens=300, temperature=0.5

start = time.time()

response = client.chat.completions.create(
    model="grok-4",
    messages=[
        {"role": "system", "content": "你是一个简洁的助手，回答不超过100字。"},
        {"role": "user", "content": "用一句话解释量子纠缠"}
    ],
    max_tokens=100,        # 强制控制输出长度
    temperature=0.3,       # 更低温度减少随机token浪费
    stream=True,
    extra_headers={
        "X-Server-Name": "限时特价"  # 选择0.6倍分组
    }
)

collected = []
for chunk in response:
    if chunk.choices[0].delta.content:
        collected.append(chunk.choices[0].delta.content)

end = time.time()
print("".join(collected))
print(f"耗时{end-start:.2f}秒，预估消耗token数：输入~1500，输出~100")


重点改动：
- **extra_headers指定分组**：直接走限时特价，费率×0.6，单次省0.02元。
- **max_tokens设小**：防止模型放飞自我，输出token直接决定费用。
- **system prompt限制长度**：成本最优化。
- **stream=True**：边生成边拿结果，大幅减少等待时间，并且云雾对流式支持无额外费用。

你只需要把这些参数组合好，每次调用就能稳稳省下0.02元。假设每天10000次调用，一个月就省6000元——够请团队吃顿大餐了。
