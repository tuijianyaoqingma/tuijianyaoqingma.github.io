如果你用Spring Boot或者任何HTTP客户端，核心代码就这么简单：

java
import okhttp3.*;
import org.json.JSONObject;

public class ClaudeClient {
    private static final String API_URL = "https://www.yunwuai.cc/v1/chat/completions";
    private static final String API_KEY = "你的[云雾api中转站](https://www.yunwuai.cc/)Key";

    public static void main(String[] args) throws Exception {
        OkHttpClient client = new OkHttpClient.Builder()
            .connectTimeout(60, TimeUnit.SECONDS)
            .readTimeout(60, TimeUnit.SECONDS)
            .build();

        JSONObject message = new JSONObject();
        message.put("role", "user");
        message.put("content", "用Java写一个冒泡排序");

        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "claude-3-5-sonnet-20241022");
        requestBody.put("messages", new JSONArray().put(message));
        requestBody.put("max_tokens", 8192);

        Request request = new Request.Builder()
            .url(API_URL)
            .header("Authorization", "Bearer " + API_KEY)
            .header("Content-Type", "application/json")
            .post(RequestBody.create(
                requestBody.toString(), 
                MediaType.parse("application/json")
            ))
            .build();

        Response response = client.newCall(request).execute();
        System.out.println(response.body().string());
    }
}


**关键点就两个：**
- `base_url` 改成 `https://www.yunwuai.cc/v1`
- `model` 参数填Claude对应的模型名（如 `claude-3-5-sonnet-20241022`）

如果你用 `openai-java` 库，那更短，直接改两个参数就完事。对Java项目来说，迁移成本几乎为零。

## Java接入时容易踩的坑（附解决方案）

我自己和身边的同事在接入过程中踩了几个雷，直接列出来，你避开就行：

**坑1：`model`参数写错**
有些教程里用旧版模型名，比如 `claude-instant-1`，实际接口早不认了。统一用最新版名称，比如 `claude-3-5-sonnet-20241022`。

**坑2：并发数过高导致报错**
官方API有速率限制，Java代码里如果并发请求太多，容易触发429错误。解决方法是在请求间加 `Thread.sleep` 控制频率，或者用线程池限制并发数。

**坑3：Token计费没算清**
Claude的Token计费包含输入和输出，有些代理只报输入价格，输出另算。[云雾api中转站](https://www.yunwuai.cc/)是**1:1按官方价格计费**，输入输出统一，不玩文字游戏。

**坑4：API key泄露**
Java代码里硬编码API key是常见错误。建议用环境变量或配置文件加密存储，避免上传到GitHub。
