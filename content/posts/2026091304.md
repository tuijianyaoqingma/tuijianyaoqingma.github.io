我们直接用 Java 演示。先加 Maven 依赖（这行不算代码，算基础设施）：

xml

</dependency>


然后，真正写代码的部分——**核心就三行**：

java
String baseUrl = "https://www.yunwuai.cc/v1";
OpenAiService service = new OpenAiService("sk-你的云雾API密钥", Duration.ofSeconds(30));
service.setBaseUrl(baseUrl);


接着就是正常调用：

java
ChatCompletionRequest request = ChatCompletionRequest.builder()
    .model("qwen-turbo")  // 直接填模型名，云雾自动路由
    .messages(List.of(new ChatMessage(ChatMessageRole.USER.value(), "用一句话介绍通义千问")))
    .build();
ChatCompletionResult result = service.createChatCompletion(request);
System.out.println(result.getChoices().get(0).getMessage().getContent());


看明白了吗？**三行代码**——创建 Service、设置 baseUrl、发出请求。之前你得去找 Qwen 的独立 Java SDK（更新慢、文档烂）、配阿里云的 AccessKey、处理非流式和流式不同的返回值结构。现在全没了。接入成本降到零，想换模型？把 `model` 参数换成 `"gpt-4o"` 或 `"claude-3-opus"` 就行。

Java 项目里常见的 `RestTemplate`、`Feign`、`WebClient` 也都能直接用——因为云雾的接口是标准的 RESTful，返回也是 OpenAI 格式的 JSON，解析起来毫无负担。
