单模型接入只能算基本功。真正体现“架构思维”的，是把这个流程泛化，做成一个优雅的多模型调度器。

来看一个简单的调度器示例：

python
import openai
from typing import List, Dict

MODEL_CONFIG = {
    "gemini": {
        "model": "gemini-2.5-pro",
        "provider": "cloud_mist"  # 你起的别名
    },
    "gpt4": {
        "model": "gpt-4o",
        "provider": "cloud_mist"
    },
    "claude": {
        "model": "claude-3-5-sonnet-20241022",
        "provider": "cloud_mist"
    }
}

client = openai.OpenAI(
    api_key="sk-你的云雾key",
    base_url="https://www.yunwuai.cc/v1"
)

def call_model(model_name: str, messages: List[Dict]):
    """调度指定模型"""
    config = MODEL_CONFIG.get(model_name)
    if not config:
        raise ValueError(f"不支持的模型: {model_name}")
    
    response = client.chat.completions.create(
        model=config["model"],
        messages=messages
    )
    return response

# 调用示例：一句话同时切换Gemini和GPT
messages = [{"role": "user", "content": "解释什么是微服务架构？"}]

# 用Gemini试试
gemini_response = call_model("gemini", messages)
print("Gemini回答:", gemini_response.choices[0].message.content)

# 秒切GPT
gpt_response = call_model("gpt4", messages)
print("GPT回答:", gpt_response.choices[0].message.content)


这套代码放在你项目的API层。想要测试某个Prompt在不同模型上的表现？改一行配置就行，不需要在代码里硬编码一堆判断逻辑。
