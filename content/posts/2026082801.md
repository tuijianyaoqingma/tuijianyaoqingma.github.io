def call_qwen(prompt: str, system_prompt: str = "你是一个有用的助手"):
    """

## 附赠：常见报错解决方案（保证不封号）

代码不是跑完就好了。你99%会遇到的弯路，我已经帮你蹚过来了。下面直接给诊断方案：

### 错误一：`401 Authentication Error`（认证失败）
**症状**：你的API请求过去，服务端显示无法验证身份。
**诊断**：
1.  **检查Key：** 你是否复制了完整的 `sk-` 开头的Key？有些平台Key会隐藏部分字符，请确认。
2.  **检查BASE_URL：** 你是否写成了 `https://www.yunwuai.cc/v1` 结尾，少了 `/v1` 或者多空格都会报错。
3.  **安全建议：** 不要在代码里硬编码Key！去[云雾ai大模型中转站](www.yunwuai.cc)后台的API key页面，**一键设置IP白名单**。这样你的Key就只有你指定的IP能调用，从根本上杜绝被泄漏、盗刷、封号的风险。

### 错误二：`404 Not Found`（接口不存在）
**症状**：请求能到达服务器，但找不到你调用的模型或接口。
**诊断**：
1.  **核对模型名：** 你写的MODEL_NAME对吗？比如通义千问的具体版本名可能叫 `qwen-max-2025-01-28`，请去云雾平台后台查看最新的模型列表。
2.  **服务状态：** 有时候部分模型会临时下线维护。直接访问云端平台状态页确认。

### 错误三：`429 Too Many Requests`（请求过限）
**症状**：批量调用时忽然卡住。
**诊断**：
1.  **速率限制：** 免费用户或低级别套餐通常有一定并发限制。在代码里，使用**指数退避重试**（exponential backoff）。
2.  **最佳实践：** 像通义千问这种模型，用 `qwen-plus` 做高频任务，用 `qwen-max` 做一次性的高质量需求，通过我的代码模板里的MODEL_NAME灵活切换。如果要大量刷数据，把 `call_qwen` 包在一个带有 `time.sleep` 的循环里。

### 错误四：`502 Bad Gateway` / `Connection Reset`（网络连接问题）
**症状**：偶尔请求超时或中断。
**诊断**：
1.  **网络：** 云雾中转站国内直连极佳，但你服务器本地的DNS或防火墙也可能有干扰。检查服务器能否 `ping` 通 `www.yunwuai.cc`。
2.  **SDK版本：** 确保你的 `openai` 库是最新版：`pip install --upgrade openai`。旧版本库的协议有时会不兼容。
3.  **最保险的写法：** 给你的请求加上重试机制。试一下这个增强版：
    python
    import time
    from tenacity import retry, stop_after_attempt, wait_exponential
    
    @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=2, max=10))
    def call_qwen_with_retry(prompt: str):
        # 直接用上面的call_qwen函数
        return call_qwen(prompt)
    
    连续失败3次就放弃，中间会自动等待。这是大厂API交互的标准玩法。

### 关于“保证不封号”的真相
市面上很多Starter总说“用了不会被封”，其实这是伪命题。封不封号完全取决于你的API Key是否只被你自己用于正规用途。

**真正让你安全的，是以下操作组合：**
1.  **从不共享Key：** 把Key从你的代码里分离出来，放在环境变量或配置文件中。
2.  **只调用“官方兼容接口”：** [云雾ai大模型中转站](https://www.yunwuai.cc/)走的是**纯AZ渠道**和**正向渠道**，不搞盗号转发，不反复切IP。
3.  **主动设置白名单：** 这是防误封的最强手段。在云雾后台API Key页面，设置**IP白名单**，只让你的服务器IP调用。哪怕别人偷了你的Key，他也用不了。
4.  **不灰产用途：** 不要用它来搞批量爬虫或营销干扰。

做到这四点，你的账号比任何人都安全。我自己的几个云端账号，连续跑了几个月，**从没收到过任何风控警告**。
