## Grok统一接入Java示例——改一行代码的事

先给你看核心代码。这个Java示例演示了如何通过[云雾ai中转站](https://www.yunwuai.cc/)的OpenAI兼容接口，统一调用Grok模型（以及其他模型）。

java
import com.theokanning.openai.service.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.time.Duration;
import java.util.Arrays;

public class UnifiedGrokClient {
    
    // [云雾ai中转站](https://www.yunwuai.cc/)统一接入地址
    private static final String BASE_URL = "https://www.yunwuai.cc/v1";
    
    // 你的云雾API Key（在官网注册获取）
    private static final String API_KEY = "your-yunwu-api-key-here";
    
    private final OpenAiService service;
    
    public UnifiedGrokClient() {
        this.service = new OpenAiService(API_KEY, Duration.ofSeconds(60));
        // 关键：设置自定义base_url
        service.setBaseUrl(BASE_URL);
    }
    
    /**
     * 统一调用Grok模型，传入模型名称和用户消息
     */
    public String callGrok(String modelName, String userMessage) {
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model(modelName)  // 例如: "grok-1" 或其他Grok模型
                .messages(Arrays.asList(
                        new ChatMessage("system", "你是Grok助手，请用中文回答。"),
                        new ChatMessage("user", userMessage)
                ))
                .maxTokens(1024)
                .temperature(0.7)
                .build();
        
        return service.createChatCompletion(request)
                .getChoices()
                .get(0)
                .getMessage()
                .getContent();
    }
    
    public static void main(String[] args) {
        UnifiedGrokClient client = new UnifiedGrokClient();
        
        // 调用Grok模型 - 低成本高性价比
        String response = client.callGrok("grok-1", "请用简洁的方式解释量子计算的基本原理");
        System.out.println("Grok回复: " + response);
    }
}


**这段代码做了什么？**

- 它用了OpenAI官方Java SDK（`com.theokanning.openai`），但把`base_url`指向了[云雾ai中转站](https://www.yunwuai.cc/)的统一接入地址。
- 你只需要传入不同模型名称（比如`grok-1`、`gpt-4o-mini`、`claude-3-haiku`），同一段代码就能调用不同模型。
- 这样做的好处：**你的业务代码不需要动，只改模型名称和base_url，就能切换到底成本模型。**
