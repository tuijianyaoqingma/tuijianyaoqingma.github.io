### 方式二：使用Apache HttpClient（100%可靠，推荐）

这个方式避开了SDK可能出现的兼容性问题，只依赖于最基础的Http请求，在任何Java环境下都能稳定运行。

**2. 编写核心代码**

java
import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.http.client.methods.HttpPost;
import org.apache.http.entity.StringEntity;
import org.apache.http.impl.client.CloseableHttpClient;
import org.apache.http.impl.client.HttpClients;
import org.apache.http.util.EntityUtils;

import java.util.List;
import java.util.Map;

public class Qwen3DirectHttpExample {

    public static void main(String[] args) throws Exception {
        // ---------- 配置 ----------
        String apiKey = "sk-your-yunwuai-api-key-here"; // 替换为你的云雾API Key
        String apiUrl = "https://www.yunwuai.cc/v1/chat/completions"; // 标准接口
        
        ObjectMapper mapper = new ObjectMapper();
        
        // ---------- 构建请求体 ----------
        Map<String, Object> requestBody = Map.of(
                "model", "qwen3", // 模型名：qwen3
                "messages", List.of(
                        Map.of("role", "system", "content", "你是一个精通Java的代码助手。"),
                        Map.of("role", "user", "content", "请用Java实现一个简单的单例模式。")
                ),
                "max_tokens", 1000,
                "temperature", 0.6,
                "stream", false // 设置为false以简化返回结果
        );
        
        String jsonBody = mapper.writeValueAsString(requestBody);
        
        // ---------- 发送请求 ----------
        try (CloseableHttpClient client = HttpClients.createDefault()) {
            HttpPost httpPost = new HttpPost(apiUrl);
            httpPost.setHeader("Authorization", "Bearer " + apiKey);
            httpPost.setHeader("Content-Type", "application/json");
            httpPost.setEntity(new StringEntity(jsonBody, "UTF-8"));
            
            String response = EntityUtils.toString(client.execute(httpPost).getEntity(), "UTF-8");
            
            // ---------- 解析结果 ----------
            Map<String, Object> result = mapper.readValue(response, Map.class);
            List<Map<String, Object>> choices = (List<Map<String, Object>>) result.get("choices");
            if (choices != null && !choices.isEmpty()) {
                Map<String, Object> message = (Map<String, Object>) choices.get(0).get("message");
                String content = (String) message.get("content");
                System.out.println("=== Qwen3回复 ===");
                System.out.println(content);
            }
        }
    }
}


**关键点解析：**
- **接口地址**：`https://www.yunwuai.cc/v1/chat/completions` 必须填写，这是云雾的通用端点。
- **模型名**：必须是`qwen3`，这是云雾内部映射的真实模型标识。如果你在后台看到其他模型名（如`qwen3:latest`），也以此为准。但绝大多数时候，直接写`qwen3`即可。
- **Header**：`Authorization: Bearer <你的Key>` 是标准身份认证，不要拼错。
- **Stream**：设置为`false`，回复会一次性返回。如果你需要流式输出，改`stream: true`并按`data:`格式解析即可。

这个代码段可以直接复制到你的Spring Boot项目或任何Java应用中，修改`apiKey`后就能运行。

## 常见问题与避坑指南

即使代码看起来很简单，实际跑起来还是有几个容易“翻车”的点，我帮你全部提前想到。

**1. 模型名写错了**
   - 错误：`model: "qwen-3"` 或 `model: "通义千问"`

**2. API Key定义错误**

**3. 网络错误 / Timeout**
   - 错误：等待时间过短，或公司网络有防火墙。
   - 正确：检查端点的可访问性。可以直接在浏览器访问`https://www.yunwuai.cc/v1/models`（带上Key），如果能返回模型列表，说明网络没问题。代码中建议设置合理的超时时间（如30秒）。

**4. 403 Forbidden**
