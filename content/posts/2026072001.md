**Java示例：ChatGPT API调用，同时轻松切换Claude**

很多Java开发者担心自己用的技术栈不支持。别怕，下面就是一个直接用Java的 `HttpClient` 调用OpenAI兼容接口的例子，并且你可以看到，切换模型只是改一个参数的事情。

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.http.HttpRequest.BodyPublishers;
import java.net.http.HttpResponse.BodyHandlers;

public class YunWuAiExample {

    // 你的云雾API Key
    private static final String API_KEY = "sk-your-yunwuai-api-key-here";
    // 云雾的Base URL
    private static final String BASE_URL = "https://www.yunwuai.cc/v1";

    public static void main(String[] args) throws Exception {
        // 调用GPT-4o
        String gptResponse = callModel("gpt-4o", "用中文告诉我，为什么API聚合平台对开发者很重要？");
        System.out.println("GPT-4o回复：" + gptResponse);

        // 调用Claude 3.5 Sonnet - 只需要改模型名称！
        String claudeResponse = callModel("claude-3-5-sonnet-20241022", "用中文告诉我，为什么API聚合平台对开发者很重要？");
        System.out.println("Claude回复：" + claudeResponse);
    }

    public static String callModel(String modelName, String prompt) throws Exception {
        HttpClient client = HttpClient.newHttpClient();

        // 构建符合OpenAI格式的请求体
        String jsonBody = String.format("""
                {
                    "model": "%s",
                    "messages": [{"role": "user", "content": "%s"}],
                    "max_tokens": 100
                }
                """, modelName, prompt);

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(BASE_URL + "/chat/completions"))
                .header("Content-Type", "application/json")
                .header("Authorization", "Bearer " + API_KEY)
                .POST(BodyPublishers.ofString(jsonBody))
                .build();

        HttpResponse<String> response = client.send(request, BodyHandlers.ofString());
        return response.body();
    }
}


你看到了吗？整个过程里，除了 `modelName` 这个字符串变了，其他所有代码——包括网络请求、鉴权方式、接口URL——完全一模一样。这就是OpenAI兼容接口的强大之处，也是[云雾ai聚合平台](https://www.yunwuai.cc/)的价值所在。
