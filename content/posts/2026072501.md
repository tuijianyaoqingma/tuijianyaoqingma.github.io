### 完整代码示例

python
from openai import OpenAI

# 1. 创建客户端，指向云雾API的接口
client = OpenAI(
    api_key="sk-你的云雾API密钥", # 将这里替换为你从云雾控制台复制的真实API Key
    base_url="https://www.yunwuai.cc/v1" # 唯一的改动点！！！

# 2. 定义你要调用的模型，这里我们使用 Grok3mini
model_name = "grok-3-mini" # 具体模型名称请以云雾平台显示为准

# 3. 构建对话消息
messages = [
    {
        "role": "system",
        "content": "你是一个顶尖的 Python 开发助手，精通代码优化和 Debug。"
    },
    {
        "role": "user",
        "content": "请用 Python 写一个快速排序算法，并解释每一步的原理。"
    }
]

# 4. 发出 API 请求，开启流式输出
print("云雾API正在调用 Grok3mini，请稍候...")
print("-" * 30)

try:
    response = client.chat.completions.create(
        model=model_name,
        messages=messages,
        temperature=0.7,  # 控制创造力
        max_tokens=4096,  # 设置最大输出长度
        stream=True        # 开启流式输出，实时获取结果
    )

    # 5. 处理流式响应
    for chunk in response:
        if chunk.choices[0].delta.content is not None:
            print(chunk.choices[0].delta.content, end="")

except Exception as e:
    print(f"请求出错：{e}")
    print("请检查你的 API Key 是否配置正确，网络是否通畅。")

print("\n" + "-" * 30)
print("Grok3mini 调用完毕！")


### 代码核心要点解析

- **避免踩坑**：如果你的 API Key 格式不正确，或者网络不通，程序会进入 `except` 分支打印错误信息，你可以根据错误信息排查。最好的方法是复制云雾后台的 Key，不要手动输入。
- **流式输出 (stream=True)**：这是推荐用法，能够让你看到大模型实时生成内容，体验更好，响应也更及时。如果你不需要流式输出，可以将其设为 `False`，然后直接打印 `response.choices[0].message.content`。
- **参数调整**：`temperature` 控制创意（0-2之间）；`max_tokens` 控制回答最大长度。你可以根据自己的需要调整。
